# Contributing to Learn AAMAD

Thank you for your interest in contributing to the Learn AAMAD course! This document provides guidelines and instructions for contributing.

---

## How to Contribute

### Reporting Issues

Found a bug, typo, or have a suggestion? Please open a GitHub issue with:
- Clear description of the issue
- Steps to reproduce (if applicable)
- Expected vs actual behavior
- Any relevant context

### Suggesting Improvements

Have an idea to improve the course? We'd love to hear it!
- Open an issue with the `enhancement` label
- Describe the improvement and why it would help learners
- Include examples if possible

### Submitting Content

Want to help write or improve course content?

1. **Fork the repository**
2. **Create a branch** for your changes (`git checkout -b feature/your-feature-name`)
3. **Make your changes** following the existing style
4. **Test your changes** (if applicable)
5. **Commit** with clear messages
6. **Push** to your fork
7. **Open a Pull Request**

---

## Content Guidelines

### Writing Style

- **Clear and concise:** Aim for clarity over complexity
- **Beginner-friendly:** Assume readers are new to agentic architecture
- **Practical:** Include examples and exercises
- **Consistent:** Follow the existing module structure

### Module Structure

Each module should include:
- Learning objectives
- Content sections
- Exercises (where applicable)
- "Check Your Understanding" section
- Next steps navigation

### Code Examples

- Use clear, commented code
- Include setup instructions
- Test all examples before submitting
- Keep examples minimal but complete

---

## Development Setup

1. Clone the repository
2. Review existing content structure
3. Make your changes
4. Preview markdown rendering
5. Submit your PR

---

## Review Process

- All contributions will be reviewed
- We may request changes or clarifications
- Once approved, your contribution will be merged

---

## Questions?

Feel free to:
- Open an issue with questions
- Reach out to maintainers
- Check existing issues/discussions
